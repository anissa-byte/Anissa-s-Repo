Before You Start

Pre-Requisites

Confirm these with your team member before scheduling the setup call:

RequirementHow to CheckMinimum
Mac computer Windows is not supported yet macOS required
Free disk space Apple menu → About This Mac → Storage 15 GB free minimum
Computer model Apple menu → About This Mac M1 chip or newer recommended
RAM Apple menu → About This Mac 8 GB minimum, 16 GB recommended
macOS version Apple menu → About This Mac Up to date recommended

Why this matters

Low disk space = dramatically slower install times. During Linnea's onboarding, the install that normally takes 7–10 minutes took 25+ minutes because her disk was nearly full. If your team member has less than 15 GB free, have them offload files to an external drive or cloud storage before the setup call.

Before the call, ask your team member to:

  • Close all unnecessary applications (browsers with lots of tabs, Spotify, etc.)
  • Have their Mac password ready (they'll need to type it during setup)
  • Accept any pending GitHub and Claude Console invites from their email

Phase 1

Download, Open & Sign In to Cursor — ~3 min

Step 1 — Download & Install

  1. Visit cursor.com in your browser
  2. Click Download (auto-detects Mac)
  3. Open the .dmg file from Downloads
  4. Drag the Cursor icon to your Applications folder
  5. Wait for the copy to finish

Step 2 — Open Cursor

  1. Open Finder → Applications
  2. Double-click Cursor
  3. If macOS shows a security prompt, click Open
  4. Skip any tutorials that pop up

If macOS blocks the app

Go to System Settings → Privacy & Security → scroll down and click "Open Anyway".

Step 3 — Sign In with Team Account

  1. Click Login when prompted
  2. Enter the team Cursor credentials
  3. Confirm login
You're done with Phase 1 when: You see the Cursor app open with a welcome screen or empty editor. You do not need to create a new project yet.

Phase 3

Let Claude Do the Rest — ~15–25 min

What to Expect

This is the longest phase. After you paste the prompt below and hit Enter:

  • It will take 7–25 minutes depending on your internet speed and computer specs
  • You will see terminal output scrolling at the bottom of the screen — this is normal
  • It may look frozen at times — if text is slowly appearing in the terminal panel, it's still working
  • The longest single step is downloading Xcode Command Line Tools — this alone can take 5–10 minutes
  • You will need to type your Mac password at one point (nothing shows as you type — that's normal)
  • Click "Yes" or "Allow" for any permission prompts that pop up

Copy and paste this entire prompt into the Claude Code chat panel:

Copy this prompt I'm a new team member setting up my machine for the first time. I need you to do the following — handle each step and confirm when it's done before moving to the next: 1. Install Homebrew (if not already installed) 2. Install Git and GitHub CLI using Homebrew 3. Install Node.js using Homebrew 4. Verify all three are installed by checking versions 5. Authenticate with GitHub using: gh auth login (use SSH, generate new SSH key, login via web browser) 6. Clone team repo: git clone git@github.com:School-of-Bots/done-for-you-division.git ~/Documents/SOB-DFY 7. Set Git identity — ask for my name and email before running the commands 8. Confirm repo cloned successfully by listing contents of ~/Documents/SOB-DFY Do each step one at a time. Tell me if anything needs my input. After EACH step is complete, say "Step X complete" clearly so I know we're moving forward. When ALL 8 steps are done, say: "ALL DONE — Setup is complete. All 8 steps finished successfully." and list the final versions of everything installed.

What It Looks Like While Running

Claude installing Homebrew. You'll see terminal output scrolling at the bottom — this is normal.
The "slow part" — Xcode Command Line Tools downloading. This step alone can take 5–10 minutes. If you see "Downloading" or "Installing Command Line Tools" in the terminal, it's still working.
The terminal will show download progress. Don't close anything — just wait.

Claude will say "ALL DONE — Setup is complete" and list the installed versions. If you're not sure, see the "Is it done?" section below.

"Is it done?" — How to Check

It's okay to ask "Is it done yet?" — Claude will tell you what's still running.

Sometimes Claude gets confused about its own progress

This actually happened during onboarding — Claude sometimes says "not done" when it actually is, or vice versa. If Claude seems confused or stuck, use these messages:

If this happens...Say this to Claude
Not sure if it's done Are all 8 steps complete? List which ones are done and which are still pending.
Claude says not done but nothing is happening Check the terminal output. Did the last command finish? What step are we actually on?
You want to verify yourself Run this and show me the output: brew --version && git --version && gh --version && node --version && ls ~/Documents/SOB-DFY
Claude is going in circles Stop. Let's verify what's actually installed. Run the version checks and show me.

When Claude needs your Mac password: type it in the terminal area at the bottom. Nothing will appear on screen as you type — this is normal macOS behavior. Just type it and press Enter.

When Claude asks for GitHub browser login: it will open your browser. Sign in, authorize, and come back to Cursor.

You're done with Phase 3 when: Claude says "ALL DONE" and you can see the SOB-DFY folder contents listed. If unsure, ask Claude: "Is setup complete? Show me the contents of ~/Documents/SOB-DFY"

Check Git

Check API Keys

Expected: Both commands print your keys. If either prints nothing, tell Claude: "My API key is not set. Help me add it to ~/.zshrc."

Test Claude Code

If a command isn't recognized

Claude Cheat Phrases — Use When Stuck

These phrases work in almost any situation where Claude seems stuck, confused, or unresponsive. Keep this page bookmarked.

Universal Rescue Phrases

"Are all steps complete? List which ones are done and which are still pending."
Nothing is happening
"Check the terminal. Is a command still running? What step are we on?"
Claude says it can't do something
"Try doing it in the terminal. Run the command directly."
Claude seems confused or looping

"Explain this error in plain English and tell me exactly what to do next."

"Start over from step [X]. Don't redo the earlier steps."
Totally lost

Known MCP Quirks

MCPs (Model Context Protocol servers) connect Claude to external tools. Some are finicky. Here's what we've learned:

Airtable

The standard airtable-mcp-server install frequently fails or reports as "not installed correctly." Use our team's custom integration instead — your admin will set this up during onboarding. The team helper script is at scripts/airtable.sh in the repo.

To test if Airtable is working:

curl -s -H "Authorization: Bearer $AIRTABLE_API_KEY" \
  "https://api.airtable.com/v0/app0MXWD2LKazuQo1/Team" | head -c 200

If you see JSON data, it's working. If you see AUTHENTICATION_REQUIRED, your token is wrong or expired.

Slack MCP

Slack MCP requires OAuth setup — it's not just an API key. It can be flaky:

  • It sometimes says "not connected" even when it's working
  • The connection can drop and need re-authentication
  • Workaround: Claude can access Slack messages through the Notion MCP instead

Don't spend more than 10 minutes troubleshooting Slack MCP during initial setup. Move on and come back to it later. It's not required for day-one work.

Context7

Generally reliable. If it fails, run npx -y @upstash/context7-mcp in the terminal to verify it installs.

Notion

Uses HTTP connection type. Usually works on first try. If it doesn't connect, verify the URL is exactly: https://mcp.notion.com/mcp

Daily Workflow Reference

Every Morning

  1. Open Cursor → SOB-DFY project
  2. Check bottom-left corner — branch must say main. If not, click it and select main.
  3. Open Terminal (Ctrl + `) and run: git pull origin main
  4. Create today's branch: click branch name → Create new branch → name it yourname-MM-DD-YY

During the Day

Work normally in Cursor. Run slash commands. Save with Cmd + S. No Git steps needed mid-day.

Every Evening

  1. Click View → Source Control
  2. Hover over Changes → click the + icon to stage
  3. Write a clear commit message (e.g., "yourname 03-11-26 — updated client context files")
  4. Click the Commit button (checkmark icon)
  5. Click Publish Branch (first time) or Sync Changes
  6. In Terminal, run: gh pr create --fill

Expected: You see a URL like https://github.com/School-of-Bots/.../pull/XX

Leadership reviews all PRs before 9 AM EST. When you pull main the next morning, approved changes are merged.

AI System Handbook

The Big Picture

What's Happening: School of Bots has built an AI agent system that automates repeatable work across all roles. Your job is evolving from task execution to quality control and client strategy using pre-built AI skills.

The One Rule: "Skills are process. Context is knowledge. Keep them separate."

Skills instruct HOW to do tasks (universal across clients). Context provides WHO it's for (client-specific). One skill + client context = scalable system.

Three Layers

  1. Skill File — Instructions (leadership maintains)
  2. Client Context — Client-specific info (GS + CW populate)
  3. Airtable — Live client data (Natasha/Anissa maintain)

Four Build Layers

  1. Layer 4: Human SOPs — How the team uses commands
  2. Layer 3: AI Skills + Slash Commands — Pre-built by leadership
  3. Layer 2: Frameworks — Repeatable structures and playbooks
  4. Layer 1: Context Library — Business knowledge and brand guides

You live in Layer 4. Leadership handles Layers 1–3.

System Evolution

  • Skills — One skill, manual invocation
  • Workflows — Multiple skills chained in sequence
  • Agents — Orchestrators deciding which skills/workflows to run
  • Scheduled Runs — Scripts running on schedule, looping through clients

Currently in Skills + Workflows stage. Human review always required.

How Slash Commands Work

  1. Open Cursor with the SOB-DFY project
  2. Open Claude Code (Cmd + I)
  3. Type your slash command with a client name: /[role]-[command] [client-name]
  4. Claude loads the skill, pulls client data from Airtable, and generates output
  5. You review — if good, act on it. If not, give Claude feedback to adjust.

Prompting Tips

Old way: Dump everything → confused output → restart.

New way: Give context in pieces → confirm understanding → next step together → iterate.

The Magic Phrase

"Before you do that, come back to me with your plan first."

  • One task per conversation — start a fresh chat for each topic
  • Be a manager, not a suggester — say "Write 3-hook DM sequence" not "Can you help me think of ideas?"
  • Give context first, confirm understanding, then instruct
  • Iterate, don't restart — if it's 70% right, tell Claude what to fix
  • Plan in Claude.ai, build in Cursor — brainstorm separately from working files

Testing & Feedback

  • Run each skill at least 2 times on 2 different clients
  • Fill the feedback form after each run (5 questions, 3 min)
  • Nothing reaches a client without leadership review
  • Production-ready = 4/5 average score, no "No" on Q1 or Q5
  • Use /feature-request for improvements or new skill ideas

Common Mistakes to Avoid

  • Don't dump too much context at once — one task per conversation
  • Don't reuse the same chat forever — fresh chat per topic
  • Don't build while brainstorming — plan in Claude.ai, build in Cursor
  • Don't skip the morning pull — always git pull origin main
  • Don't write vague commits — use specific messages
  • Don't edit skills directly — use /feature-request
  • Don't assume AI is right — review every output before acting on it

FAQ

Claude, Claude Code, Cursor — what's the difference?

Brain, brain with hands, the office.

I'm not a developer. Can I do this?

Yes. You're running slash commands and reviewing outputs. The system is designed for non-developers.

Do I build skills?

Not now. Use /feature-request for ideas.

Why not one skill per client?

Maintenance nightmare. One skill + client context = scalable. That's the model.

What if a skill gives bad output?

Don't use it. Fill the feedback form. The skill gets updated. That's the testing cycle.

How do I know which commands to use?

Open slash-command-library.md in Cursor. Your lead also assigns starting commands during onboarding.

What if the install takes longer than 25 minutes?

Check your free disk space (Apple menu → About This Mac → Storage). If you're low on space, that's the bottleneck. Close unnecessary apps and clear space. If storage is fine and it's been 30+ minutes, start a fresh chat and paste the prompt again.
